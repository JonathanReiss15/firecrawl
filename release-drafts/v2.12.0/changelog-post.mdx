## v2.12.0 is live

Firecrawl v2.12.0 lands a `search` monitor target that runs SERP queries on a schedule and alerts on new or materially-developed results (with a per-page LLM judge stack, goal-versioned dedupe, and flat deterministic billing), a first-party Slack App with a `/monitor` slash command and Block Kit monitor alerts, a deterministic `menu` scrape format gated behind `menuBeta`, and a hosted parse upload flow so MCP and remote tools can call `/v2/parse` on local files without exposing API keys. It also adds a `totalPages` field on PDF scrapes for truncation detection, an `enterprise` search parameter across the JS/Python/Ruby/.NET SDKs, `webhook` support on JS extract, a `SELF_HOSTED_DOMAIN` env var to fix response URLs behind reverse proxies, transparent gzip decompression on Fire Engine scrapes, an Express 5.2 + TypeScript 7 RC build, new NuQ FoundationDB metrics under a new admin endpoint, and finishes the migration to Autumn as the single source of truth for credit accounting across v0, v1, and v2.

### Highlights

- **`search` monitor target** â€” Added a first-class `search` monitor target accepting `queries[]`, `searchWindow`, `includeDomains[]`, `excludeDomains[]`, and `maxResults`, with `raw` (no LLM) and `deep` (LLM judge) depths derived from `judgeEnabled`, a `material_dev` alert mode that re-alerts on material developments of a known event, and stable opaque event keys to prevent label-drift duplicates. Exposed across the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs with a `MonitorSearchTargetResult` type.
- **Search monitor billing** â€” Search monitors now bill at a flat per-check rate: `SEARCH_CREDITS` is charged `ceil(totalResults / 10) * 2`, judged results are charged 1 credit per result (down from 5), and `raw` mode pays nothing for the judge. Deep-mode per-page scrapes flow through the shared scrape pipeline as queued jobs billed under `CREDITS`, not double-counted on the search check.
- **Slack App** â€” Connect a Slack workspace and receive monitor alerts as Block Kit summaries, or drive monitors from Slack with the `/monitor` slash command (`watch`, `list`, `cancel`, `status`, `help`). OAuth, signed webhooks, encrypted bot-token storage, auto-join of public channels, and a `notification.slack` block on monitors are all included.
- **`menu` scrape format** â€” A new `formats: ["menu"]` returns deterministic structured restaurant-menu data as `document.menu`, mirroring the existing `product` format end-to-end. Beta-gated via the `menuBeta` team flag with a `MENU_EXTRACTION_SERVICE_URL` self-host kill-switch, available across the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs.
- **Hosted `/v2/parse` upload flow** â€” `POST /v2/parse/upload-url` mints a short-lived, team-scoped `uploadRef` plus signed storage upload parameters; the client uploads bytes directly to GCS; then `POST /v2/parse` accepts JSON with `uploadRef` and parses normally. Refs are HMAC-signed with a 10-minute expiry and bound to the team, object path, filename, content type, and max size, so MCP clients can parse local files without exposing an API key to the model or shell.
- **PDF `totalPages` metadata** â€” PDF scrapes now return `metadata.totalPages` alongside `numPages`, so callers can tell a truncated result (`totalPages > numPages`) from a full parse. Threaded through the API, index cache, and JS/Python SDK types.
- **`enterprise` search parameter across SDKs** â€” The `/v2/search` `enterprise` option (e.g. `["zdr"]`, `["anon"]`) now works from JS, Python, Ruby, and .NET (`@mendable/firecrawl-js@4.29.0`, `firecrawl@4.31.0` Python, `firecrawl@1.10.0` Ruby, `firecrawl-sdk@1.9.0` .NET). Previously JS/Python silently dropped the field and .NET typed it as `bool?`.
- **JS SDK `webhook` on extract** â€” `startExtract` and `extract` in the JS SDK now accept `webhook?: string | WebhookConfig | null`, matching `crawl` and `batchScrape`.
- **`SELF_HOSTED_DOMAIN` env var** â€” Self-hosted instances behind a reverse proxy can now set `SELF_HOSTED_DOMAIN` (or send `X-Forwarded-Host`) so crawl status URLs, batch-scrape URLs, parse-upload links, and pagination `next` links use the externally-reachable host instead of the internal bind address.
- **Transparent gzip decompression** â€” Fire Engine scrapes now detect gzipped payloads by magic bytes and transparently decompress them (up to 50 MB), so callers no longer have to pre-inflate `.gz` documents.
- **Express 5 and TypeScript 7 RC** â€” The API and worker now run on Express 5.2.1 with `express-ws` reapplied to the v1/v2 routers, and the API build compiler is `typescript@7.0.1-rc` for roughly 3Ă— faster clean builds (median ~1.84 s vs ~5.60 s on TS 5.9.2). `knip` continues to run on TS 5 through a wrapper to avoid runtime API breaks.
- **NuQ FoundationDB metrics** â€” New per-status FDB queue gauges (`queued`, `active`, `backlog`, `completed`, `failed`) and a `firecrawl_nuq_fdb_pending_jobs` worker-load gauge, served from a new `GET /admin/<BULL_AUTH_KEY>/nuq-fdb-metrics` Prometheus endpoint for FDB-based queue autoscaling.
- **`nuq-postgres` multi-arch image** â€” The `nuq-postgres` image now builds and publishes for both `linux/amd64` and `linux/arm64`, matching the rest of the stack.
- **Autumn is the single source of truth for credits** â€” Removed the last ACUC-based credit accounting paths from the v0/v1/v2 hot path (including the `fireclaw` multi-play check and the unverified agent-key 50-credit cap), and pointed ACUC RPCs at `v50` â€” giving single-team orgs `GREATEST(override, plan)` concurrency, pay-as-you-go orgs a concurrency floor of 5, and merging per-team `rate_limits` overrides with the base plan per-endpoint.
- **Lazy-loaded images** â€” Fixed the largest-image picker to fall back to `data-srcset` and `data-src` when `srcset` is missing, so below-the-fold images on lazy-loading sites come back with the real URL instead of a 5px placeholder.
- **`proxy: enhanced` / `stealth`** â€” Fixed `proxy: enhanced` and `proxy: stealth` scrapes silently routing through non-stealth engines on the cloud â€” the fallback list is now restricted to stealth-capable engines when stealth is requested.
- **Playwright cookie redirect fix** â€” Playwright-engine scrapes now seed the `Cookie` header into the browser context via `context.addCookies()` scoped to the registrable domain, so authenticated pages that 302 (including cross-subdomain hops) no longer come back as the login page.
- **CVE patches** â€” Resolved multiple CVEs by upgrading `multer` to `2.2.0` (GHSA-3p4h-7m6x-2hcm, GHSA-72gw-mp4g-v24j), `undici` to `7.28.0` with a pnpm override locking it major-bound (GHSA-35p6-xmwp-9g52, GHSA-g8m3-5g58-fq7m, GHSA-hm92-r4w5-c3mj, GHSA-p88m-4jfj-68fv, GHSA-pr7r-676h-xcf6, GHSA-vmh5-mc38-953g, GHSA-vxpw-j846-p89q), and `js-yaml` to `3.15.0` via a major-bound pnpm override (GHSA-h67p-54hq-rp68), plus removing the malicious `serpapi-python==0.1.5` pin from the GPT-4.1 company researcher example in favor of `google-search-results`.
